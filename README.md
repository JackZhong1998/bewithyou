# bewithyou - 声音克隆陪伴应用版本v1

一个基于 React + TypeScript 的声音克隆应用，使用 Inworld AI 的语音克隆和 TTS 服务。

## 功能特性

- 🎤 **声音克隆**：上传音频文件克隆声音
- 🌍 **多语言支持**：支持中文简体、中文繁体、韩文、日语、泰语、越南语
- 📚 **背单词功能**：使用克隆的声音学习英语单词
- 👤 **角色管理**：创建和管理多个声音角色
- 📱 **响应式设计**：完美适配手机端和电脑端

## 技术栈

- React 18
- TypeScript
- Vite
- Tailwind CSS
- Lucide React Icons
- Inworld AI API

## 快速开始

### 安装依赖

```bash
npm install
```

### 配置环境变量

复制 `.env.example` 为 `.env` 并填入你的配置：

```env
# 必填：Inworld
VITE_INWORLD_API_KEY=your_api_key_here
VITE_INWORLD_WORKSPACE=workspaces/your_workspace

# 可选：邀请码
VITE_INVITE_CODES=520,123,456
VITE_INVITE_FORM_URL=https://your-form-url

# 登录与数据库（可选，不配置则仅使用本地存储）
VITE_CLERK_PUBLISHABLE_KEY=pk_test_xxxx    # Clerk 登录
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key
```

- **Clerk**：在 [clerk.com](https://clerk.com) 创建应用，将 Publishable Key 填入上述变量；未配置时右上角登录入口仍显示，但点击会无效果，角色数据仅存本地。
- **Supabase**：在 [supabase.com](https://supabase.com) 创建项目后，在 SQL Editor 中执行 `supabase/migrations/001_create_characters.sql` 创建 `characters` 表；将项目 URL 与 anon key 填入 `.env`。配置后，已登录用户的角色会同步到数据库，且「社区角色」Tab 会从库中拉取已分享角色。

### 启动开发服务器

```bash
npm run dev
```

### 构建生产版本

```bash
npm run build
```

## 主要功能说明

### 声音克隆

1. 上传音频文件（支持 WAV、MP3、M4A，最大 10MB）
2. 选择原声语言
3. 选择是否去除背景噪音
4. 同意用户协议
5. 输入邀请码（首次使用）
6. 开始克隆

### 背单词

- 显示单词和中文含义
- 三个英文例句（暧昧情书风格）及中文翻译
- 点击播放按钮使用克隆声音朗读
- 支持重读和下一个单词

### 角色管理

- 创建多个声音角色
- 自定义角色名称和头像
- 查看角色列表

## 环境变量说明

- `VITE_INWORLD_API_KEY`: Inworld AI API Key（Base64 编码）
- `VITE_INWORLD_WORKSPACE`: Inworld 工作空间 ID（格式：workspaces/xxx）
- `VITE_INVITE_CODES`: 邀请码列表，用逗号分隔（如：520,123,456）
- `VITE_INVITE_FORM_URL`: 获取邀请码的表单链接

## 注意事项

1. 音频文件建议 5-15 秒，清晰无背景噪音
2. 首次使用需要输入邀请码
3. 邀请码会保存在本地，后续使用无需再次输入
4. 所有数据保存在浏览器本地存储中

## 产品需求文档（PRD）

### 一、产品概述

**产品名称**：待定（建议：EchoTalk / 声伴 / VoiceMate）

**产品定位**：基于AI声音克隆的沉浸式英语陪伴学习应用

**核心价值主张**：让用户用自己喜爱/熟悉的声音（偶像、朋友、虚拟恋人等）陪伴学习，通过情感连接提升学习坚持度与效果。

**目标用户**：
- 18-35岁英语学习者
- 追星族/二次元爱好者（希望用偶像/角色声音学习）
- 异地恋/社交需求用户（希望用恋人/朋友声音陪伴）
- 对英语学习感到枯燥、难以坚持的人群

---

### 二、功能模块详细需求

#### 模块1：角色定制系统

**1.1 角色创建页**

| 功能项 | 详细说明 |
|--------|----------|
| 入口 | 首页"创建角色"按钮 / 角色列表页"+"按钮 |
| 上传方式 | 支持上传视频文件（提取音频）或纯音频文件 |
| 支持格式 | MP4、MOV、MP3、WAV、M4A（限制：视频≤100MB，音频≤20MB，时长30秒-5分钟） |
| 原声语言选择 | 下拉选择：中文、英语、日语、韩语等（用于优化克隆效果） |
| 克隆处理 | 上传后显示处理进度，预计1-3分钟完成 |
| 结果预览 | 克隆完成后提供试听功能（朗读一段示例文本） |

**1.2 角色信息编辑**

| 字段 | 规则 |
|------|------|
| 角色名称 | 非必填，默认："我的角色#001"（递增编号） |
| 角色头像 | 非必填，默认：系统生成的抽象声波图案或首字母头像 |
| 角色描述 | 非必填，默认："这是一个通过语音克隆创建的角色" |

---

#### 模块2：角色管理

**2.1 角色列表页**

- **Tab切换**：我的角色 | 社区角色
- **我的角色**：
  - 泳道横向列表展示，显示头像、名称、描述、创建时间
- **社区角色**：
  - 网格列表展示其他用户分享的角色
  - 筛选：最热/最新/按标签筛选
  - 点击可使用（克隆到自己的角色库，仅复制配置，不占用克隆额度）

**2.2 角色详情页**

| 功能 | 说明 |
|------|------|
| 信息展示 | 大图头像、名称、描述、创建者|
| 试听按钮 | 播放角色示例语音 |
| 编辑按钮 | （仅自己创建的角色）跳转编辑页 |
| 分享开关 | 开启后角色出现在社区，关闭后仅自己可见 |
| 开始学习 | 快捷入口，直接进入背单词页并选中该角色 |
| 删除角色 | 二次确认后删除 |

---

#### 模块3：背单词学习系统（核心功能）

**3.1 学习设置页（选择角色后进入）**

| 设置项 | 说明 |
|--------|------|
| 学习目标 | 选择单词书（四级/六级/雅思/托福/自定义等） |
| 母语语言 | 下拉选择：中文、日语、韩语等（影响AI讲解的语言） |
| 讲解角色模式 | **系统预设角色** 或 **自定义角色描述** |
| 自定义角色描述（当选择自定义时） | 文本输入，例如："你是我在国外留学的男朋友，声音温柔，会耐心纠正我的发音，偶尔撒娇" |

**3.2 学习流程**

```
页面结构：沉浸式全屏，背景为角色头像模糊大图
┌─────────────────────────────┐
│  [单词]  [音标]  [收藏⭐]    │
│                             │
│      [角色头像/动画]         │
│      （点击可重播语音）       │
│                             │
│   "嗨，今天我们来学abandon~  │
│    这个词意思是'放弃'，记住   │
│    哦，不要abandon学习呀❤️"   │
│                             │
│   [例句展示区]               │
│   [AI生成的角色化讲解文案]    │
│                             │
│   [认识 😊]  [模糊 🤔]  [不认识 😵] │
└─────────────────────────────┘
```

**3.3 AI讲解内容生成逻辑**

**输入**：
- 单词：abandon
- 词义：放弃；抛弃
- 例句：He abandoned his car in the snow.
- 角色设定：一套角色提示词如温柔恋人/严厉老师/暧昧对象/搞笑朋友/霸道总裁等根据选项或用户自定义的内容，作为变量输入进来

**输出要求**：
- 语气符合角色设定
- 包含：母语解释 + 发音技巧 + 记忆方法 + 例句朗读+生成符合人设的例句，比如暧昧对象
- 长度：30-60秒语音
- 可加入角色化元素（昵称、互动话术、情感表达）

**示例输出**：
> "宝贝，今天学的abandon可不能真的abandon掉哦～来跟我读，a-ban-don，重音在第二个音节。我绝对不会abandon你的，对吧？这个词意思是'放弃'"

**3.4 学习记录**
- 记录每个单词的学习次数、掌握程度、最后学习时间
- 根据艾宾浩斯遗忘曲线推送复习
